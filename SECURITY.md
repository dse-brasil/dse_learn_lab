# 🔒 Política de Segurança (SECURITY.md)

Esta política descreve como o **DSE.LearnLab** lida com a segurança do sistema e como você pode relatar vulnerabilidades de forma responsável.

---

## 🛠️ Versões Suportadas

Atualmente, apenas a versão em desenvolvimento ativo na branch `main` é ativamente monitorada para vulnerabilidades de segurança. À medida que lançarmos versões estáveis (v1.0+), atualizaremos a tabela abaixo:

| Versão | Suportada | Notas |
| :--- | :--- | :--- |
| **v0.x (dev)** | Sim | Desenvolvimento ativo. Correções são aplicadas diretamente na branch `main`. |
| **< v0.1** | Não | Versões experimentais legadas não recebem patches. |

---

## 📬 Como Relatar uma Vulnerabilidade

Se você descobrir uma vulnerabilidade de segurança no DSE.LearnLab, **por favor, não abra uma issue pública no GitHub**. Em vez disso, relate o problema de forma privada seguindo estes passos:

1.  Envie um e-mail detalhado para **seguranca-dse@exemplo.com** (e-mail provisório para fins acadêmicos/open-source).
2.  No e-mail, inclua:
    *   Uma descrição detalhada do problema de segurança.
    *   Passos para reproduzir a vulnerabilidade (incluindo código de exploração (PoC), se aplicável).
    *   O impacto potencial caso a vulnerabilidade seja explorada.
    *   Caso deseje, seu nome ou pseudônimo para fins de crédito na correção.

Prometemos responder a todos os relatórios de vulnerabilidade de segurança dentro de **48 horas**, fornecendo uma estimativa de prazo para a resolução do problema. Manteremos você informado sobre o progresso até que a correção seja publicada.

---

## 🛡️ Declaração de Porto Seguro (Safe Harbor)

Incentivamos a pesquisa de segurança responsável em nosso sistema. Se você fizer um esforço de boa-fé para cumprir esta política durante sua pesquisa de segurança, nós:

*   Consideraremos sua pesquisa autorizada.
*   Não iniciaremos ou apoiaremos ações legais relacionadas à sua pesquisa.
*   Trabalharemos com você para entender e resolver o problema rapidamente.
